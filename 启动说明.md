# 开发服务器启动说明

## 问题原因
Windows HPC Pack 的 `node.exe` 与 Node.js 的 `node.exe` 冲突，即使 Node.js 在 PATH 最前面，系统仍可能使用 HPC Pack 的 node.exe。

## 解决方案

### 方法 1：使用批处理文件（最简单，推荐）

直接双击 `start-dev.bat` 文件，或在命令行运行：
```cmd
start-dev.bat
```

**优点：**
- 不需要设置执行策略
- 直接使用完整路径调用 npm
- 最可靠的方式

### 方法 2：使用 PowerShell 脚本

在 PowerShell 中运行：
```powershell
.\start-dev.ps1
```

如果遇到执行策略错误，先运行：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

然后再次运行：
```powershell
.\start-dev.ps1
```

### 方法 3：使用完整路径（手动）

在 PowerShell 或 CMD 中直接运行：
```powershell
& "C:\Program Files\nodejs\npm.cmd" run dev
```

或：
```cmd
"C:\Program Files\nodejs\npm.cmd" run dev
```

## 诊断工具

如果脚本无法运行，可以运行诊断脚本查看问题：
```powershell
powershell -ExecutionPolicy Bypass -File test-node.ps1
```

## 注意事项

1. 如果 Node.js 安装在其他路径，需要修改脚本中的 `$nodejsPath` 或 `NODEJS_PATH` 变量
2. 脚本使用完整路径调用 npm，避免 PATH 冲突
3. 批处理文件（.bat）通常比 PowerShell 脚本更可靠
